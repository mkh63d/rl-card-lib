# Library reference runs

Committed full run records for the **bundled example games** (Klondike, Macao),
loaded by [`library_reference_store()`](../reference.py) and folded into a report
by `HtmlReport.build(reference_store=...)`.

They let someone experimenting with their own agent compare against the library's
learners on an example game without retraining those learners first. The report
adds them **only for a game the user is already reporting on**, so a custom game
— which has no runs here — never shows them.

## Layout

This directory is a read-only [`RunStore`](../../../report/src/rl_card_lib/report/run_record.py):

```
reference_data/
├── models/
│   └── <game>__<agent>/run.json     # one full RunRecord per library learner
└── baselines/
    └── <game>.json                  # the non-learning baseline rows per game
```

## Regenerating

These are the machine output of a sweep over the bundled games and their
learners:

```bash
python packages/examples/scripts/run_sweep.py --games klondike,macao
```

then copy the resulting `results/models/<game>__<agent>/run.json` and
`results/baselines/<game>.json` here. Only `run.json` and the baseline JSON are
kept — figures are regenerated by the report, and checkpoints are not shipped.
